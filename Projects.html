Particle Simulations with Cellular Automata

This project was coded from scratch with Java in the Eclipes IDE using the Java Swing library for it's graphics. My goal for this personal project was to create a particle simulation sandbox game with realistic interactions betweeen a various different elements. I accomplished what you see in the video using a modified version of the "Conway's Game of Life" where the state of any tile each update is determinded by the state of it's neighbors. As an example, a yellow tile represents sand. Each update a sand tile will first check if the tile below it is empty, if so it updates it self to swap it's state with the tile below. Additional rules can be applied such as 'If the tile below is also sand, try falling randomly to the left or right'. Very simple rules such as these, while simple in concept, can cause a number of different patterns to emerge, and in this case, it does a good job of simulating particles in a 2D plane in real time.

Survive the Hoard!

This project, similar to the project above, was coded in Java with the Eclipes IDE. This projected initially started as a test to see how I could go about simulating a large number of complex moving entities and it ended up turning into an arcade style 'survive the hoard' style game. Starting the game you are pursued by a small number of green square(or zombies) and it is your task to avade and defeat the zombies to earn coins and survive as long as you can. The twist is that the numbers of enemies around you increases at an exponential rate and you must fight to survive.

Managing the memory of the an infinitely expanding hoard of simulated entites was the main challenge of this project. I settled with an array that would recyle defeated enemies back into the pool of enemies to be spawned. This achieved the desired goal of creating the appearance of a never-ending and increasingly dense hoard endlessly pursing you with only a finite amount of memory being used. This game is what showed me the power of Object Oriented Programming as I was easily able to reuse the same sections of code from the enemy entities for making the coin entities that similarly needed to appear in large amounts and without end all while using the least amount of memory.
